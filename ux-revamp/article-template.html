Your Article Title Goes Here, Even When It Spans Two Lines

A concise, compelling subtitle that frames the argument. One or two sentences, no more. Pulls the reader in before the byline.

Joel Sooriah Head of Product · happyapps
12 min read

This is the opening paragraph — your lede. Write it with force. One strong claim, a provocation, or an image that demands the reader keep going. No hedging, no warming up. The first sentence has to land.

Normal body copy follows. Lora at a comfortable reading size with generous line-height. References can be footnoted1 using simple anchor links. Bold for emphasis, italic for a term being introduced or for a voice shift.

Section One: The Problem

Each section opens with a paragraph that sets up what follows. Don't start with "In this section…" — just make the first claim. The structure should be visible in the prose, not announced.

More body copy. Link to external sources with inline anchor text rather than bare URLs. Keep links purposeful — every outbound link is a small exit ramp.

The spec is not a deliverable. It is a shared commitment to what done looks like — before a single line of code is written.

Continuation after the pull quote. The quote should feel earned — place it after you've built up enough context for it to resonate, not as a way to break up density.

A nuanced sub-point

Use h3 sparingly — only when a section genuinely has distinct sub-arguments worth naming. Overuse dilutes the hierarchy. Three levels of heading (h1 → h2 → h3) is usually the ceiling. 

// Inline code block for technical content
const spec = { source: "truth", status: "before-code" };
// Use sparingly — only when the exact syntax matters

Section Two: The Argument

Second major section. Notice the left-rule accent on the heading — it creates a strong vertical rhythm as the reader scans. Each h2 should be a claim, not just a label. "Why specs fail" beats "Spec failures."

Key concept

Use the callout block for a definition, a framework summary, or a short concept that deserves visual separation from the main argument flow. Keep it to three sentences maximum.

Resume the main argument after the callout. The callout is a brief stop, not a detour — the reader should drop back into the thread immediately.

Closing: What This Means

The conclusion should do three things: restate the core claim without repeating yourself, draw out the implication the reader should carry away, and close on a line that feels final. No "In conclusion…" — just the last move in the argument.

End on something concrete. A question that opens the next problem. A call to action. A provocation. Something that gives the reader a reason to share.
